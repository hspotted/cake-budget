# Contributing

## Branching
- `production`: protected, PR-only
- `development`: default branch for feature work

## Commits
- Use English and Conventional Commits (see COMMIT_CONVENTIONS.md)
- Keep subject <= 72 chars

## PRs
- Link an issue
- Pass CI (Biome + tests)
- Keep descriptions concise; use bullet points

## Docs
- Update `README.md` and `CHANGELOG.md`
- Place additional docs under `docs/`
